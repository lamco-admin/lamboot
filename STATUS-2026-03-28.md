## What LamBoot Is

A memory-safe UEFI bootloader written in Rust, targeting hypervisor-managed VM environments (Proxmox) while remaining usable as a general-purpose boot manager. Licensed MIT/Apache-2.0.

Differentiators: graphical boot menu with mouse support (unique among Rust bootloaders), NVRAM-based crash recovery with automatic fallback, pre-boot diagnostic modules, and Proxmox host-side boot health monitoring via NVRAM variables readable from OVMF_VARS.

## What's Been Done (Phases 1-4 from MVP spec)

The technical assessment from 2026-03-27 identified 9 critical blockers (B1-B9) and 8 significant shortcomings (S1-S8). The subsequent development pass addressed all of them:

### Critical Blockers — All Resolved

| ID | Issue | Resolution |
|----|-------|------------|
| B1 | Broken TOML parser | Rewritten with section-aware parsing, array support |
| B2 | Legacy Linux boot non-functional | Implemented LoadFile2 initrd delivery (kernel 5.7+) |
| B3 | Crash counter without fallback | Full state machine: assess_boot_health(), fallback selection, counter clear |
| B4 | No Secure Boot | Shim integration (ShimLock protocol), signing toolkit |
| B5 | No BLS support | Full Type 1 parser with 14 fields, UAPI.10 sort, boot counting |
| B6 | No kernel update integration | BLS compatibility = kernel-install works automatically |
| B7 | No filesystem drivers | Loadable EFI driver support, ships ext4/btrfs from rEFInd |
| B8 | Single-disk only | Multi-partition scanning of all SimpleFileSystem handles |
| B9 | TOML sections broken | Subsumed by B1 fix |

### Significant Shortcomings — All Resolved

| ID | Issue | Resolution |
|----|-------|------------|
| S1 | Per-pixel rendering | Double-buffered framebuffer with single BltOp::BufferToVideo |
| S2 | Incomplete font | Full VGA 8x8 ROM font (256 characters) |
| S3 | Mouse hardcoded 1920x1080 | Dynamic resolution from GOP mode query |
| S4 | No timestamps | uefi::runtime::get_time() integration |
| S5 | Initrd not loaded | LoadFile2 protocol with LINUX_EFI_INITRD_MEDIA_GUID |
| S6 | Counter never cleared | OS-side clear via Boot Loader Interface variables |
| S7 | No TPM | TCG2 protocol — PCR 4 (kernel), 5 (config), 12 (cmdline) |
| S8 | No network boot | Deferred — not in scope for VM-focused MVP |

### Additional Features Implemented

- Boot Loader Interface variables (LoaderInfo, LoaderEntrySelected, LoaderBootCountPath) for systemd compatibility
- systemd-compatible boot counting (+N[-M] filename suffixes, works with systemd-bless-boot)
- Proxmox NVRAM monitoring tool (lamboot-monitor.py)
- Serial console fallback for headless VMs
- ARM64 build support
- Signing toolkit (tools/sign-lamboot.sh)

## Strategic Context

LamBoot sits in **Layer 4** (Proxmox RDP ecosystem) of the Lamco Development Strategy. It provides a lightweight, monitorable UEFI bootloader for VMs managed by Proxmox.

### Competitive Position

| | LamBoot | GRUB | systemd-boot | Sprout | rEFInd |
|--|---------|------|-------------|--------|--------|
| Language | Rust | C | C | Rust | C++ |
| Binary | 112KB | ~4MB | ~300KB | ~200KB | ~1MB |
| GUI | Yes (mouse) | Text | Text | Text | Yes (icons) |
| BLS | Type 1 | Partial | Type 1+2 | Type 1 | No |
| Crash Recovery | State machine | No | Boot counting | No | No |
| TPM | PCR 4/5/12 | No | PCR 4/5/8/12 | Roadmap | No |
| VM Focus | Primary | Generic | Generic | Cloud | Generic |

**Primary competitor:** Sprout (edera-dev/sprout) — also Rust/uefi-rs, but targets cloud-native, no GUI, no crash recovery. LamBoot's differentiators are exactly Sprout's gaps.

**Market timing:** GRUB stagnating, distributions migrating to systemd-boot. Window of opportunity for alternatives with clear differentiation.
